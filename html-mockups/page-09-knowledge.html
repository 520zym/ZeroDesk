ZeroDesk 项目开发规范 v3.0

本文档定义了 ZeroDesk 多 Agent 协同工作台项目的核心开发规范，所有参与协作的 Agent 和人类成员均须遵守以下约定。

一、接口通信规范

所有 Agent 间通信采用 JSON-RPC 2.0 协议，消息体必须包含 task_id、sender、timestamp 三个必填字段。响应超时阈值统一设置为 30 秒，超时后自动触发重试机制，最大重试次数为 3 次。

二、命名与代码风格

前端组件使用 PascalCase 命名，工具函数使用 camelCase。后端 API 路由统一使用 /api/v1/ 前缀，资源命名采用复数形式。所有变量和函数命名需具备语义化，禁止使用无意义的缩写。

三、文档与版本管理

  • 每次需求变更必须同步更新知识库中的对应文档
  • 版本号遵循语义化版本规范 (SemVer)，重大变更递增主版本号
  • Agent 自动生成的文档需标注来源和置信度评分
  • 人工审核通过后方可将文档状态标记为"已确认"
