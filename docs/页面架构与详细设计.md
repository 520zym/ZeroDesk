# ZeroDesk 页面架构与详细设计

## 1. 设计目标与范围

本文件定义 ZeroDesk 桌面端的完整页面信息架构，覆盖：

- 页面清单与职责边界
- 每个页面的核心元素与可执行操作
- 页面之间的关系与主流程
- MVP 裁剪范围

设计原则：

- **职责单一**：一个页面只做一类核心决策
- **AI 自主执行**：用户定义目标和约束，AI 自行规划执行步骤与协作方式
- **全程可控**：任务可暂停、可重试、可人工接管
- **结果可追溯**：配置、决策、产出有记录
- **低认知负担**：结构稳定、路径短、操作少

---

## 2. 全局信息架构（IA）

一级导航分组（5 组）：

- **任务**：任务中心、执行控制台、任务历史
- **组织**：Agent 管理、团队管理
- **能力**：模型与路由、Skills 中心
- **资产**：知识库、Prompt/模板中心
- **分析**：数据看板与系统设置

全局固定组件：

- 顶栏：全局搜索、通知、全局暂停、工作区切换
- 左侧主导航：上述 5 组
- 右侧上下文抽屉：当前选中对象详情（Agent/Skill/任务节点）

---

## 3. 页面清单（11 页 + 1 过渡页）

### 页面 01：工作区入口

**目标**：进入应用并选择本地工作区。
**核心元素**：最近工作区列表、创建/导入按钮、环境自检（数据库/模型连接）。
**可执行操作**：打开、创建、导入、修复。
**输出**：当前工作区上下文（路径、配置、最近任务）。
**页面关系**：进入后跳转 → 页面 02。

---

### 页面 02：任务中心（含任务发起表单）

**目标**：查看任务全貌、发起新任务。
**核心元素**：
- 任务列表与统计卡片（进行中/已完成/失败/草稿）
- 状态筛选与搜索
- 快速模板入口
- **新建任务面板**（弹出式轻量表单）：
  - 目标输入（自然语言描述）
  - 执行方案选择：AI 智能规划 / 复用已有团队
  - 成本控制（经济/标准/高质量/不限四档）
  - 高级设置（折叠）：质量门禁、重试策略、超预算策略、执行超时
  - 启动按钮 → 跳转到计划视图

**可执行操作**：新建任务、打开执行中任务、复制历史配置、删除草稿。
**输出**：任务目标 + 约束 → 跳转页面 03（计划视图）。
**页面关系**：启动任务 → 页面 03（计划视图）→ 确认后 → 页面 04；查看历史 → 页面 11。

---

### 页面 03：执行计划视图（过渡页，不在侧边栏导航中）

**目标**：展示 AI 生成的执行计划，允许用户审核和调整后再执行。
**核心元素**：
- 目标回显 + 成本档位标签
- **AI 规划说明**：自然语言解释规划理由，标注 Agent 来源（"从你的 Agent 库中匹配到 2 个，另建议新建 1 个"）
- **左侧主区域 — 执行步骤**（做什么）：
  - 每步显示：步骤编号、步骤名称（可编辑）、操作描述（可编辑）、分配的 Agent（引用）、输出传递方式
  - 步骤只定义"做什么"和"谁做"，不包含 Agent 的能力配置
  - 可拖拽排序、删除步骤、添加新步骤
- **右侧侧边栏 — Agent 团队**（谁来做）：
  - Agent 卡片列表，每个显示名称、模型、工具权限、来源标签（复用/新建）
  - 点击 Agent 卡片展开编辑：Prompt、模型、工具权限
  - 复用 Agent 的修改仅作用于本次任务，不影响 Agent 库原始配置
  - 新建 Agent 默认勾选"任务完成后沉淀到 Agent 库"
  - "添加 Agent"按钮：从已有 Agent 库选人 / 手动创建新 Agent
- **预估成本**：复用/新建计数、Token、费用
- "保存为团队模板"按钮：将当前 Agent 组合保存到团队管理页面
- "重新规划"按钮：让 AI 重新生成 Plan
- "确认执行"按钮：进入执行控制台

**关键设计决策**：
- 这是一个 **过渡页面**，不在左侧导航栏中——只能从新建任务流程进入
- **步骤与 Agent 分离**：步骤是"做什么"，Agent 是"谁来做"，两者通过引用关联
- **AI 优先复用用户已有的 Agent**，只在库中没有合适人选时才建议新建
- 新建的 Agent 任务完成后沉淀到 Agent 库，成为可复用资产——保护用户的 Agent 资产投入
- Plan 就是执行方案本身，不需要归类为某种命名的"模式"

**可执行操作**：编辑步骤描述、增删排序步骤、重新分配 Agent、编辑 Agent 配置、添加 Agent、保存为团队模板、重新规划、确认执行。
**输出**：确认后的 Plan → 跳转页面 04。
**页面关系**：来自 → 页面 02；确认后 → 页面 04；Agent 沉淀 → 页面 05；保存模板 → 页面 06。

---

### 页面 04：执行控制台（核心）

**目标**：实时观测执行过程与人工接管。
**核心元素**：
- 左栏：参与 Agent 状态面板（头像、状态指示、Token 用量）
- 中央：消息流（Agent 对话气泡、系统通知、数据产出）
- 右栏：运行指标（耗时、Token、费用、节点进度）
- 顶部控制栏：暂停、继续、重试、终止、导出快照
- 底部：人工指令输入框

**关键设计决策**：
- AI 按用户确认的 Plan 执行，执行过程实时呈现
- 用户可在任意时刻暂停并修改 AI 的决策

**可执行操作**：暂停/恢复、修改节点输出、插入纠偏指令、从失败节点续跑、导出快照。
**输出**：执行日志、中间产物、最终产物、失败诊断。
**页面关系**：执行完成 → 页面 11；模型异常 → 页面 07。

---

### 页面 05：Agent 管理

**目标**：管理 Agent 配置（每个 Agent 是一个带特定 Prompt 和权限的 LLM 调用配置）。
**核心元素**：Agent 卡片网格、详情面板（Prompt 编辑器、模型选择、工具权限、私有 Skills）。
**可执行操作**：新建/复制/归档、调 Prompt、调权限、AI 智能拆解、保存为模板。
**输出**：Agent Profile。
**页面关系**：Skills 来自 → 页面 08；Prompt 版本 → 页面 10。

---

### 页面 06：团队管理

**目标**：管理多 Agent 组织结构与共享能力。
**核心元素**：团队卡片、成员关系、共享 Skills 池、团队结构编辑器。
**可执行操作**：组建团队、批量授权、保存团队模板。
**输出**：Team Profile（成员 + 共享能力）。
**页面关系**：成员来自 → 页面 05；被任务中心调用 → 页面 02。

---

### 页面 07：模型与路由

**目标**：管理模型供应商与可用模型池，配置容灾降级与容错机制。
**核心元素**：Provider 卡片（含健康检查）、模型池表格（质量/速度/价格）、Fallback 链、容错设置（重试/退避/预算护栏）。
**可执行操作**：新增/编辑/测试模型源、配置 Fallback 链、配置重试/退避、设置预算护栏。
**输出**：可用模型清单 + Fallback Policy + Resilience Policy。
**页面关系**：影响 → 页面 04 的每次推理。

**关键设计决策**：
- **模型选择发生在 Agent 层面**，不在本页面。本页面只管理"有哪些模型可用"和"模型不可用时怎么兜底"。
- 系统 AI 根据 Agent 的 Prompt 特征与任务类型，自动从模型池推荐最合适的模型（推荐逻辑参考模型的质量等级、速度、价格）。
- 用户可在 Agent 配置页面（页面 05）或计划视图（页面 03）手动覆盖 AI 的推荐。
- 本页面不包含"路由规则"——不需要用户手动定义"什么类型任务用什么模型"，这由系统 AI 自主判断。

---

### 页面 08：Skills 中心

**目标**：统一管理本地与在线 Skills。
**核心元素**：已安装 Skills、市场搜索、版本管理、作用域分配（全局/团队/Agent）。
**可执行操作**：安装、升级、锁版本、回滚、分配、禁用/卸载。
**输出**：Skill Registry + Scope Binding。

---

### 页面 09：知识库

**目标**：沉淀公共知识与步骤级记忆，支持检索注入。
**核心元素**：文档目录树、文档编辑器、标签、检索配置、版本时间线。
**可执行操作**：编辑、设置可见性、调整检索阈值、回滚版本。
**输出**：可注入知识片段（含来源与版本）。

---

### 页面 10：Prompt/模板中心

**目标**：管理 Prompt 资产与可复用任务模板。
**核心元素**：Prompt 版本历史、Diff 视图、稳定版标记、任务模板库。
**可执行操作**：版本对比、回滚、模板参数化、模板发布。
**输出**：Prompt Baseline + Workflow Templates。

---

### 页面 11：任务历史与复盘

**目标**：历史复用与故障复盘。
**核心元素**：历史任务表格、失败路径可视化、重试记录、导出报告。
**可执行操作**：复用历史配置、导出报告、关联审计记录。
**输出**：复盘报告（成本/质量/失败原因）。

---

### 页面 12：数据看板与系统设置

**目标**：持续优化与系统治理。
**核心元素**：KPI 看板（完成率、成本、时延、返工率）、密钥管理、权限白名单、审计导出。
**可执行操作**：按维度筛选、定位瓶颈、调整策略、导出审计数据。
**输出**：优化建议与系统治理记录。

---

## 4. 页面关系与主流程

### 流程 A：从零到首次跑通
`01 工作区入口` → `02 任务中心（新建任务）` → `03 计划视图（审核/调整 Plan）` → `04 执行控制台` → `11 任务历史`

### 流程 B：能力扩展
`08 Skills 中心` → `05/06 绑定能力` → `02 发起任务` → `04 执行验证`

### 流程 C：持续优化
`12 数据看板` → `07 模型路由` / `10 Prompt 中心` → `02 发起任务` → `04 再执行`

---

## 5. 关键数据对象

- **Task**：任务目标、状态、预算、执行摘要
- **Run Record**：执行日志、AI 规划的步骤、重试链路、人工干预记录
- **Agent Profile**：提示词、模型、工具权限、私有 Skills
- **Team Profile**：成员关系、共享 Skills
- **Skill Registry**：来源、版本、权限、作用域绑定
- **Knowledge Item**：文档内容、标签、版本、可见范围
- **Policy**：路由策略、降级策略、预算护栏

---

## 6. 任务生命周期

统一状态：
- `draft`（草稿）→ `ready`（可执行）→ `running`（执行中）
- `running` ↔ `paused`（已暂停）
- `running` → `blocked`（等待人工确认）→ `running`
- `running` → `failed`（失败）/ `completed`（完成）
- `completed` → `archived`（归档）

约束：
- `failed` 必须记录失败节点与失败类型
- `completed` 必须具备最小复盘数据（输入、输出、成本、耗时）

## 8. 跨页面一致性规则

- 同名 Skill 优先级：`Agent 私有 > 团队共享 > 全局安装`
- 任何任务都必须支持：暂停、终止、重试、续跑
- AI 的所有自主决策（步骤规划、模型路由、降级）都在 Plan 中可解释、可调整、可追溯
- 高权限能力必须显式授权（安装前 + 运行时）
- 超预算必须降级或请求人工确认，不得静默继续
