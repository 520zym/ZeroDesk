# ZeroDesk 技术栈

## 总体架构

```
┌───────────────────────────────────────────────────────┐
│                     Tauri 2.x Shell                   │
│  ┌───────────────────┐  ┌───────────────────────────┐ │
│  │   Frontend (Web)   │  │     Backend (Rust)        │ │
│  │                   │  │                           │ │
│  │  React 19         │  │  Tauri 2.x Runtime        │ │
│  │  TypeScript 5.x   │  │  SQLite (sqlx)            │ │
│  │  Tailwind CSS 4   │  │  reqwest + SSE Stream     │ │
│  │  shadcn/ui        │  │  Stronghold (密钥)        │ │
│  │  Zustand          │  │  tokio (异步运行时)        │ │
│  │  TanStack Query   │  │                           │ │
│  │  React Router     │  │                           │ │
│  └───────────────────┘  └───────────────────────────┘ │
│            ↕ Tauri IPC (invoke / event)               │
└───────────────────────────────────────────────────────┘
```

前后端通信基于 Tauri IPC：

| 机制 | 方向 | 用途 |
|---|---|---|
| **invoke**（命令调用） | 前端 → Rust | 请求-响应。获取任务列表、保存 Agent 配置、启动任务等 |
| **event**（事件推送） | Rust → 前端 | 实时推送。Agent 消息流、Token 消耗更新、任务状态变更等 |

## 一、桌面框架层

| 选型 | 说明 |
|---|---|
| **Tauri 2.x** | Rust 后端 + 系统 WebView 渲染前端。包体积 ~5-8MB，内存占用低 |
| **WebView2**（Windows）/ **WKWebView**（macOS） | 系统自带的 Web 渲染引擎，Tauri 自动调用，不打包 Chromium |

平台兼容性：

| 平台 | 最低支持版本 | 原因 |
|---|---|---|
| Windows | Windows 10 (1803+) | WebView2 最低要求 |
| macOS | macOS 11 (Big Sur) | WKWebView 现代 CSS 特性需要 Big Sur+ |

## 二、前端技术栈

### 核心框架

| 选型 | 版本 | 作用 | 选型理由 |
|---|---|---|---|
| **React** | 19 | UI 框架 | 生态最大，Tauri 官方模板直接支持，shadcn/ui 依赖 React |
| **TypeScript** | 5.x | 类型系统 | Tauri 可从 Rust 自动生成 TS 类型定义，前后端类型打通 |
| **Vite** | 6.x | 构建工具 | Tauri 默认集成，HMR 快，构建快 |

### 样式与 UI 组件

| 选型 | 作用 | 选型理由 |
|---|---|---|
| **Tailwind CSS 4** | 原子化样式 | 通过 theme 配置映射莫兰迪设计系统的 CSS 变量 |
| **shadcn/ui** | 组件库 | 组件源码直接复制到项目中，完全可控，可逐个组件适配莫兰迪风格 |
| **Radix UI** | 底层无样式原语 | shadcn/ui 的底层依赖，处理焦点管理、键盘导航、无障碍等 |

shadcn/ui 不是 npm 依赖，而是源码级组件集合——拥有完整控制权，适合深度定制设计系统。

### 状态管理与数据层

| 选型 | 作用 | 选型理由 |
|---|---|---|
| **Zustand** | 全局 UI 状态 | 管理纯前端状态：当前工作区、侧边栏折叠、全局暂停等 |
| **TanStack Query** | 异步数据管理 | 管理所有从 Rust 后端获取的数据，自带缓存、自动刷新、乐观更新 |

分工原则：
- Zustand → 纯前端 UI 状态（侧边栏展开/折叠、弹窗开关、当前选中的 Tab）
- TanStack Query → 所有后端数据（通过 Tauri invoke 调用 Rust 命令获取）

### 路由

| 选型 | 作用 |
|---|---|
| **React Router 7** | 11 个页面 + 过渡页的 SPA 路由管理 |

### 功能性依赖

| 选型 | 用在哪 |
|---|---|
| **@dnd-kit/core** | 计划视图（页面 03）的步骤拖拽排序 |
| **recharts** 或 **ECharts** | 数据看板（页面 12）的图表可视化 |
| **@tiptap/react** | 知识库（页面 09）的富文本 / Markdown 编辑器 |
| **diff-match-patch** | Prompt 版本对比（页面 10）的文本 diff |
| **cmdk** | 全局命令面板（`Ctrl+K` / `⌘K` 唤起） |
| **sonner** | 轻量 Toast 通知 |
| **date-fns** | 日期格式化（"10 分钟前"、时间线等） |

## 三、Rust 后端技术栈

### 核心运行时

| 选型 | 作用 |
|---|---|
| **tokio** | 异步运行时，Tauri 2.x 内置依赖 |
| **serde** + **serde_json** | 序列化，前后端 IPC 数据传输的基础 |

### 数据库

| 选型 | 作用 | 选型理由 |
|---|---|---|
| **sqlx**（SQLite driver） | 数据库访问 | 异步、编译期 SQL 检查、内置迁移工具 |
| **sqlx-cli** | 迁移管理 | `sqlx migrate add/run` 管理 schema 版本演进 |

选 sqlx 的原因：
- vs `rusqlite`：rusqlite 是同步的，sqlx 异步且有编译期 SQL 检查
- vs `sea-orm`：ZeroDesk 大量 JSON 列操作、FTS5、sqlite-vec 需要原生 SQL，ORM 反而碍事
- vs `diesel`：不支持异步，编译速度慢

### SQLite 扩展

| 选型 | 作用 |
|---|---|
| **FTS5** | SQLite 内建全文检索引擎，知识库关键词搜索 |
| **sqlite-vec** | 向量检索扩展，知识库 RAG 语义搜索 |

### LLM API 通信

| 选型 | 作用 |
|---|---|
| **reqwest** | HTTP 客户端，向 LLM API 发请求 |
| **reqwest-eventsource** | SSE 流式响应解析，Agent 消息流式输出 |
| **tiktoken-rs** | Token 计数，实时消耗统计与预算控制 |

### 安全存储

| 选型 | 作用 |
|---|---|
| **tauri-plugin-stronghold** | 加密存储 API Key 等敏感凭据 |

### 其他 Rust 依赖

| 选型 | 作用 |
|---|---|
| **uuid** | 生成唯一 ID |
| **chrono** | 时间处理 |
| **tracing** + **tracing-subscriber** | 结构化日志 |
| **thiserror** / **anyhow** | 错误处理 |
| **notify**（crate） | 文件系统监听，知识库文件变更时自动同步 |

## 四、数据存储架构

核心原则：**能用 SQLite 解决的全用 SQLite**，只在密钥存储和全局配置上用专项方案。

```
ZeroDesk/
├── config.json                          ← 全局配置（tauri-plugin-store）
├── secrets.stronghold                   ← 加密密钥（tauri-plugin-stronghold）
└── workspaces/
    └── my-project/
        ├── workspace.db                 ← SQLite 主库（WAL 模式）
        │   ├── tasks, runs, steps       ← 结构化业务数据
        │   ├── agents, teams, skills    ← 资产配置
        │   ├── knowledge_items          ← 知识库文档 + 版本
        │   ├── knowledge_fts (FTS5)     ← 全文检索索引
        │   ├── knowledge_vectors (vec0) ← 向量检索索引
        │   ├── execution_messages       ← 执行日志与消息流
        │   ├── prompt_versions          ← Prompt 版本历史
        │   └── audit_log               ← 审计日志
        └── files/                       ← 知识库附件、导出快照等大文件
```

一个工作区 = 一个 SQLite 文件 + 一个附件目录。

SQLite 配置要点：
- **WAL 模式**：任务执行时多 Agent 并发写日志，读写不互斥
- **JSON 支持**：Agent Profile 等半结构化配置用 JSON 列存储，保持灵活性的同时支持 SQL 查询
- **版本化数据**：追加版本行模式（不用 Git），回滚 = 插入新版本
